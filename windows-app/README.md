# WEB2POS Table Order - Windows App

테이블오더 디바이스용 윈도우즈 데스크톱 애플리케이션입니다.

## 주요 기능

- 🔑 **디바이스 고유 ID**: 자동 생성되어 POS에서 개별 디바이스 식별
- 📡 **POS 자동 등록**: 서버 연결 시 자동으로 POS에 등록
- 💓 **Heartbeat**: 30초마다 상태 전송 (온라인/오프라인 감지)
- 🔄 **테이블 자동 배정**: POS에서 테이블 배정 시 자동 적용
- 🌐 **WebView**: 테이블 오더 웹페이지 표시
- 🖥️ **시스템 트레이**: 백그라운드 실행 지원

## 설치 및 실행

### 개발 모드
```bash
cd windows-app
npm install
npm start
```

### 빌드 (설치 파일 생성)
```bash
# 설치 프로그램 (.exe)
npm run build:win

# 포터블 버전 (설치 불필요)
npm run build:portable
```

빌드된 파일은 `dist` 폴더에 생성됩니다.

## 설정 방법

1. **앱 실행**: 앱을 실행하면 설정 화면이 표시됩니다.

2. **POS 서버 주소 입력**: 
   - 예: `http://192.168.1.100:3088`
   - "Test" 버튼으로 연결 테스트

3. **테이블 번호 입력**:
   - POS에서 표시되는 테이블 ID 입력 (예: T1, A3)
   - 또는 POS에서 배정된 테이블 사용

4. **저장 및 시작**:
   - "Save & Start" 버튼 클릭
   - 테이블 오더 페이지가 자동으로 로드됩니다.

## 시스템 트레이

- 앱을 닫으면 시스템 트레이에서 계속 실행됩니다.
- 트레이 아이콘 클릭: 창 표시
- 트레이 아이콘 우클릭:
  - Show: 창 표시
  - Settings: 설정 화면 열기
  - Quit: 앱 종료

## 디바이스 ID

- 각 디바이스에 고유 ID가 자동 생성됩니다.
- 형식: `WIN-XXXXXXXX`
- 설정 리셋해도 ID는 유지됩니다.
- POS의 "Table Devices" 화면에서 디바이스 관리 가능

## POS 연동

### 디바이스 관리 (POS)
1. POS 백오피스 → Hardware Manager → Table Devices
2. 등록된 디바이스 목록 확인
3. 테이블 배정/해제
4. 온라인/오프라인 상태 모니터링

### 테이블 배정 워크플로우
1. 윈도우 태블릿에서 앱 실행 및 POS 서버 연결
2. 디바이스가 POS에 자동 등록됨
3. POS에서 테이블 배정
4. 태블릿에 자동으로 테이블 배정 적용
5. 테이블 오더 페이지 자동 로드

## 요구 사항

- Windows 10 이상

- POS 서버와 동일 네트워크

## 문제 해결

### 연결 실패
- POS 서버 주소 확인 (http:// 포함)
- 방화벽 설정 확인
- POS 서버가 실행 중인지 확인

### 테이블 배정 안됨
- POS에서 디바이스가 "Registered"인지 확인
- POS에서 테이블 배정 후 잠시 대기 (최대 10초)

### 아이콘 생성
빌드 시 `.ico` 파일이 필요합니다. SVG를 ICO로 변환:
- https://convertio.co/svg-ico/
- 256x256 크기로 변환
